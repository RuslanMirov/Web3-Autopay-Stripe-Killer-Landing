The Problem

Web3 payments are broken
for recurring use cases.

Every existing solution requires either unlimited token approvals, custodian trust, or active user involvement for each billing cycle. None of these are acceptable for a financial product.

Today's Reality
How Web3 subscriptions work right now
  • Unlimited ERC-20 approvals — single exploit drains entire wallet
  • Users must sign every transaction — no true automation
  • Custodial "pull payment" services hold private keys on your behalf
  • Off-chain cron jobs — payments can be blocked, manipulated, or stopped
  • No standardized subscription state — each dApp reinvents the wheel
  • Zero interoperability between platforms or billing periods
AutoPay3
What AutoPay3 changes
  • Tokens held in self-custody smart wallet — zero external approvals
  • Services pull payment automatically via ERC-4337 EntryPoint — no user action
  • Subscription state lives entirely on-chain — transparent, auditable, immutable
  • Hard caps on amount and interval enforced by contract — no overcharging possible
  • One deposit, any number of services — shared wallet, isolated subscriptions
  • Any EVM-compatible chain — deploy once, scale everywhere

Competitive Moat

Four compounding
advantages.

Network effects, switching costs, and a developer community that grows wider the more dApps integrate — each advantage reinforces the others.

I
Shared Subscriber Pool
Every user who creates a funded AutoPay3 wallet can instantly subscribe to any integrated service — no re-onboarding, no new wallet, no new deposit. The network value grows with each new dApp that joins. Users are shared infrastructure, not siloed per-app.
II
Developer SDK Lock-In
Once a dApp ships subscription flows on the AutoPay3 SDK, replacement requires re-engineering wallet creation, subscription state, gas abstraction, and billing UI. The full stack integration creates meaningful switching cost — not just API key churn.
III
Open Source as Distribution
Open sourcing the core protocol is a deliberate go-to-market choice. Auditable contracts are a prerequisite for enterprise fintech and DAO treasury integrations. Every fork is a referral. Every contributor is a potential customer. Trust compounds over time.
IV
Builder Community Flywheel
Early integrating dApps receive co-marketing access, shared user distribution, and co-branded billing portals. This creates a network of advocates who promote AutoPay3 as part of their own growth — turning each new integration into a distribution event.
