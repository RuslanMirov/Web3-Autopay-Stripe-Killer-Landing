The Problem

Web3 has payments.
It doesn't have billing.

Every existing approach to recurring Web3 payments forces a trade-off: either give up custody, hand over a private key, or manually approve every single charge. None are production-ready for real subscriptions.

Today's Reality

How Web3 "subscriptions" work today

  • Unlimited ERC-20 approvals — one exploit drains the entire wallet
  • Users must sign every transaction — no real automation
  • Custodial services hold private keys to execute pulls
  • Off-chain bots — payments can fail, be blocked, or manipulated
  • No standard subscription state — every dApp reinvents the wheel
AutoPay3

What changes with AutoPay3

  • Tokens stay in your self-custody smart wallet — zero external approvals
  • ERC-4337 EntryPoint pulls automatically — no user action ever needed
  • Subscription state is entirely on-chain — immutable and auditable
  • Hard caps on amount and interval enforced in contract — no overcharging
  • One deposit, any number of services — shared wallet, isolated subs
